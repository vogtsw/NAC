# NAC Web控制台使用指南

## 🌐 前端交互界面

NAC提供了一个简单易用的Web控制台，可以通过浏览器与NAC系统进行交互。

## 🚀 快速开始

### 1. 安装依赖

```bash
pnpm install
```

### 2. 配置环境变量

确保 `.env` 文件已配置好API密钥：

```bash
ZHIPU_API_KEY=your_api_key_here
```

### 3. 启动Web服务

```bash
pnpm web
```

### 4. 访问控制台

打开浏览器，访问：**http://localhost:3000**

### 核心功能

#### 1. **任务输入**
- 在文本框中输入任务描述
- 点击"执行任务"按钮提交
- 支持自然语言描述

#### 2. **快捷任务**
预置了常用任务模板：
- **GitHub热搜** - 总结GitHub热门AI项目
- **代码分析** - 分析代码质量
- **生成API** - 生成RESTful API
- **生成测试** - 生成单元测试

#### 3. **执行结果**
- 实时显示任务执行结果
- 支持Markdown格式化
- 显示任务指标（耗时、成功率等）
- 可导出结果为文本文件

#### 4. **执行日志**
- 实时显示系统日志
- 彩色分级显示（INFO/WARN/ERROR/SUCCESS）
- 自动滚动到最新日志

#### 5. **系统信息**
- 可用Agent数量
- 可用Skill数量
- 运行中任务数
- 已完成任务数

## 🎯 使用示例

### 示例1: 搜索GitHub热门项目

1. 点击"GitHub热搜"快捷按钮
2. 任务描述自动填充
3. 点击"执行任务"
4. 查看结果面板显示的项目列表

### 示例2: 自定义任务

1. 在输入框输入：`分析以下代码的性能瓶颈并提供优化建议`
2. 点击"执行任务"
3. 系统自动：
   - 解析意图
   - 选择合适的Agent
   - 构建DAG
   - 并行执行任务
   - 返回分析结果

### 示例3: 代码生成

1. 输入：`创建一个TypeScript用户认证类，包含登录、注册、密码加密功能`
2. 点击"执行任务"
3. 查看生成的代码和测试用例

## ⚙️ 高级配置

### 修改端口

编辑 `src/api/server.ts`:

```typescript
this.config = {
  port: 3000,  // 修改为你想要的端口
};
```

### 启用HTTPS

生产环境建议使用反向代理（如Nginx）提供HTTPS。

### 自定义样式

编辑 `web/styles.css` 自定义界面样式。

## 🐛 故障排查

### 无法连接到服务器

1. 检查API服务是否启动：
   ```bash
   curl http://localhost:3000/health
   ```

2. 检查端口是否被占用：
   ```bash
   netstat -ano | findstr :3000
   ```

3. 查看日志输出

### 任务执行失败

1. 查看执行日志了解错误详情
2. 检查API密钥是否配置正确
3. 确认LLM服务是否可用

### 界面显示异常

1. 清除浏览器缓存
2. 检查浏览器控制台错误
3. 确认静态文件路径正确

## 📝 API文档

### 提交任务

```http
POST /api/v1/tasks/submit
Content-Type: application/json

{
  "user_input": "你的任务描述",
  "session_id": "可选会话ID",
  "context": {}
}
```

### 获取技能列表

```http
GET /api/v1/skills
```

### 获取Agent列表

```http
GET /api/v1/agents
```

### WebSocket连接

```javascript
const ws = new WebSocket('ws://localhost:3000/ws');

ws.send(JSON.stringify({
  type: 'task',
  userInput: '你的任务描述',
  sessionId: 'session-id'
}));
```

## 🔧 开发模式

### 前端开发

前端文件位于 `web/` 目录：

- `index.html` - 主页面
- `styles.css` - 样式文件
- `app.js` - 应用逻辑

### 修改后重新加载

前端是纯静态文件，修改后刷新浏览器即可。
